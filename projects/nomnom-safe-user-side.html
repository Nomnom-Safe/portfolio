Ownership and scope

I independently developed the cross-platform MVP of the consumer-facing application using Flutter and Dart within a Scrum cadence. The implementation pairs with the business-side data model so diners can trust what they see when filtering for allergens or dietary preferences.

Product highlights

  • Centralized discovery of restaurant-supplied allergen and dietary metadata.
  • Sorting and filtering workflows tuned for quick answers in real dining contexts.
  • Integration with the broader capstone system vision—structured data instead of static PDF menus.

Collaboration

While the consumer MVP was an individual engineering effort, it sits inside a collaborative capstone program: shared research, prototyping, REST and database integration, and continuous iteration with teammates building the business-facing React experience.

Repository layout includes Flutter application sources, Firebase configuration, Cloud Functions, and web marketing assets—see the README inside the repo for run instructions.
